## Chapter 8: Building Smart Assistants: Agentic AI and Conversational AI — Flashcards

| # | Front (Question) | Back (Answer) |
|---|-----------------|---------------|
| 1 | What is Agentic AI? | AI systems that can autonomously perform complex, multi-step tasks by combining different AI capabilities and tools to achieve goals without constant human intervention |
| 2 | What distinguishes Agentic AI from traditional AI assistants? | Agentic AI can plan, reason, use tools, and execute multi-step workflows autonomously, while traditional assistants typically respond to single queries |
| 3 | What is tool use in the context of AI agents? | The ability of AI agents to interact with external systems, APIs, databases, and services to accomplish tasks beyond their core language capabilities |
| 4 | What is Azure Bot Service? | A Microsoft cloud platform for building, deploying, and managing intelligent conversational bots across multiple channels |
| 5 | What are the fundamentals of Conversational AI? | Natural language understanding (NLU), dialog management, natural language generation (NLG), and context maintenance across conversation turns |
| 6 | What is Question Answering in Azure? | A feature that extracts answers from semi-structured content like FAQs, manuals, and documents using natural language queries |
| 7 | What are the key components needed to build an FAQ bot? | A knowledge base of question-answer pairs, natural language understanding, and a bot framework to handle conversations |
| 8 | What is orchestration in multi-agent solutions? | The coordination and management of multiple AI agents working together, routing tasks to appropriate agents and combining their outputs |
| 9 | What channels can Azure Bot Service deploy to? | Microsoft Teams, web chat, Slack, Facebook Messenger, SMS, email, and other communication platforms |
| 10 | What is a complex workflow in Agentic AI? | A series of interconnected tasks that require planning, tool selection, execution, error handling, and adaptation to achieve an end goal |
| 11 | How do AI agents handle multi-step tasks? | By breaking down goals into subtasks, selecting appropriate tools for each step, executing actions, evaluating results, and iterating as needed |
| 12 | What is the role of memory in conversational AI? | To maintain context across conversation turns, remember user preferences, and provide coherent, contextually relevant responses |
| 13 | What is intent recognition in conversational AI? | Identifying the user's purpose or goal from their natural language input to determine the appropriate response or action |
| 14 | Why is orchestration important for multi-agent solutions? | It ensures agents work together efficiently, prevents conflicts, manages dependencies, and delivers cohesive results to users |
| 15 | What makes Agentic AI the "next evolution" in AI? | Its ability to move beyond reactive responses to proactive, autonomous task completion with reasoning and planning capabilities |

### Key Terms

| Term | Definition |
|------|-----------|
| Agentic AI | AI systems capable of autonomous decision-making and executing complex multi-step tasks using various tools |
| Tool Use | An AI agent's ability to invoke external services, APIs, or functions to complete tasks |
| Azure Bot Service | Microsoft's cloud platform for building and deploying conversational AI bots |
| Question Answering | Azure AI capability that extracts answers from documents and knowledge bases using natural language |
| Orchestration | The coordination layer that manages multiple AI agents and routes tasks appropriately |
| Intent | The goal or purpose a user expresses through their natural language input |
| Dialog Management | The component that controls conversation flow and determines appropriate responses |

### Memory Tricks
- **AGENT = Autonomous Goal-oriented ENtity with Tools** - Remember that agents work toward goals using tools autonomously
- **Bot Service Channels = "Teams Slack Facebook" (TSF)** - Think "Text Sent Fast" for the main deployment channels
- **Q&A builds FAQ** - Question Answering service is your go-to for FAQ bots
- **Orchestration is like a conductor** - Just as a conductor coordinates musicians, orchestration coordinates multiple agents to create harmony
